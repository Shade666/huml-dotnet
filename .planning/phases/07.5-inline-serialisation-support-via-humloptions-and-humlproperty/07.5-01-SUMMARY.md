## Decisions Made
- `InlineMode` enum instead of `bool?` on `HumlPropertyAttribute.Inline` — C# requires attribute named argument types to be non-nullable (`CS0655`); attribute constructor parameters also cannot be `bool?` (`CS0181`). The `InlineMode.Inherit = 0` default achieves the same semantic as `null`.
- `PropertyDescriptor.Inline` remains `bool?` — the switch expression `InlineMode → bool?` at build time keeps the serialiser dispatch simple and aligns with the plan's original `bool?` contract for `PropertyDescriptor`.
- `InlineMode` in its own `InlineMode.cs` file — Meziantou analyzer `MA0048` enforces one-type-per-file rule (`TreatWarningsAsErrors` is active).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Changed HumlPropertyAttribute.Inline from bool? to InlineMode enum**
- **Found during:** Task 2 (creating test file)
- **Issue:** Plan specified `public bool? Inline { get; init; }` on `HumlPropertyAttribute`. C# forbids `bool?` as a named attribute argument type (`CS0655`) and also as an attribute constructor parameter (`CS0181`). The test POCOs in the plan used `[HumlProperty(Inline = true)]` syntax which is illegal.
- **Fix:** Introduced `InlineMode` enum (`Inherit=0`, `Inline=1`, `Multiline=2`) in a separate `InlineMode.cs` file. Changed `HumlPropertyAttribute.Inline` to type `InlineMode` with default `InlineMode.Inherit`. `PropertyDescriptor.Inline` remains `bool?` — converted via switch expression at cache build time. Test POCOs use `[HumlProperty(Inline = InlineMode.Inline)]`.
- **Files modified:** `HumlPropertyAttribute.cs`, `InlineMode.cs` (new), `PropertyDescriptor.cs`, `HumlSerializerInlineTests.cs`
- **Verification:** `dotnet build` exits 0 with no warnings; all 17 contract/green tests pass
- **Committed in:** `69d2336` (Task 2 commit)
