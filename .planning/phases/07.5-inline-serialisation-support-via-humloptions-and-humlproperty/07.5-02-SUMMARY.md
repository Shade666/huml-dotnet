## Accomplishments
- Extended `EmitEntry` with `bool? inlineOverride = null` parameter; computes `wantInline = inlineOverride ?? (options.CollectionFormat == CollectionFormat.Inline)`
- Added `EmitInlineSequence` helper: emits `key:: v1, v2, v3\n` for scalar-only lists
- Added `EmitInlineDictionary` helper: emits `key:: k1: v1, k2: v2\n` for scalar-valued dicts

- Wired `desc.Inline` through `SerializeMappingBody` so per-property `[HumlProperty(Inline = InlineMode.Inline/Multiline)]` overrides the global setting
- `SerializeDictionaryBody` passes `inlineOverride: false` to `EmitEntry` so dict body entries always emit multiline (inline is a POCO-property-level concern, not recursive)
- All 21 inline tests pass green; 686 total tests green across net8.0/net9.0/net10.0 (no regressions)
- Created `docs/inline-serialisation.md` with 148-line usage guide covering global option, per-property override, precedence rules, scalar fallback, empty collections, and version compatibility

## Decisions Made
- `SerializeDictionaryBody` passes `inlineOverride: false` to `EmitEntry` — when the outer dict falls back to multiline (because its values are non-scalar), the inner scalar lists within the dict body should also emit multiline. The `Serialize_InlineOption_ComplexDict_FallsBackToMultiline` test specifies this behavior explicitly. Inline is a POCO-property-level decision; dict entries do not carry per-entry `PropertyDescriptor` metadata, so they cannot have per-entry overrides.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] SerializeDictionaryBody must pass inlineOverride: false to EmitEntry**
- **Found during:** Task 1 (running inline tests after initial implementation)
- **Issue:** After implementing inline dispatch, `Serialize_InlineOption_ComplexDict_FallsBackToMultiline` failed. The outer `Dictionary<string, List<int>>` correctly fell back to multiline (values are lists, not scalar), but then `SerializeDictionaryBody` called `EmitEntry` without `inlineOverride`, causing the inner `List<int>` entries to be emitted inline (they are scalar-only). The test specified `  key::\n` (multiline).
- **Fix:** Changed `SerializeDictionaryBody` to pass `inlineOverride: false` to `EmitEntry` for all dict body entries. Inline is a POCO-property-level concern; dict body entries have no `PropertyDescriptor` and should not inherit the global `CollectionFormat.Inline` setting.
- **Files modified:** `src/Huml.Net/Serialization/HumlSerializer.cs`
- **Verification:** `dotnet test --filter "FullyQualifiedName~HumlSerializerInlineTests"` — 21/21 pass; `dotnet test` — 686/686 pass
