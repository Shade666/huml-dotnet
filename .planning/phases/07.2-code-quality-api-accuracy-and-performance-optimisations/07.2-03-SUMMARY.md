# Phase 07.2 Plan 03: HumlInlineMapping AST Disambiguation Summary

**HumlInlineMapping introduced as a distinct AST node for inline/empty mapping blocks, disambiguating them from root HumlDocument; deserializer dispatches both node types through a shared DeserializeMappingEntries helper.**

## Accomplishments

### Task 1: HumlInlineMapping node and parser production sites (D-04)

- Created `src/Huml.Net/Parser/HumlInlineMapping.cs` — `public sealed record HumlInlineMapping(IReadOnlyList<HumlNode> Entries) : HumlNode` mirroring HumlDocument but extending HumlNode directly (no shared abstract base).
- Updated `HumlParser.ParseInlineDict()` return type from `HumlDocument` to `HumlInlineMapping`.
- Updated `RootType.InlineDict` branch to wrap `ParseInlineDict()` result: `new HumlDocument(inlineMapping.Entries)` — root always returns HumlDocument.
- Updated `RootType.EmptyDict` branch to use `HumlInlineMapping` for the inner node instead of `HumlDocument`.
- Updated `ParseInlineVectorValue()` EmptyDict branch to return `new HumlInlineMapping(Array.Empty<HumlNode>())`.
- Updated `ParseMultilineList` EmptyDict branch to return `new HumlInlineMapping(Array.Empty<HumlNode>())`.
- Added `HumlInlineMapping` to `All_concrete_types_are_assignable_to_HumlNode` polymorphism test.
- Added 3 new `HumlNodeTests`: `HumlInlineMapping_is_sealed_and_derives_from_HumlNode`, `HumlInlineMapping_exposes_Entries`, `HumlInlineMapping_structural_equality`.

### Task 2: Deserializer dispatch and test updates (D-05)

- Extracted `DeserializeMappingEntries(IReadOnlyList<HumlNode> entries, Type targetType)` from `DeserializeDocument` body — shared helper for both HumlDocument and HumlInlineMapping paths.
- Refactored `DeserializeNode` to dispatch both `HumlDocument` and `HumlInlineMapping` to `DeserializeMappingEntries`.
- Refactored `DeserializeDictionary` to accept `IReadOnlyList<HumlNode>` instead of `HumlDocument` — called from `DeserializeMappingEntries`.
- Updated `HumlParserTests.Parse_InlineEmptyDict_ReturnsEmptyDocument` (renamed to `_ReturnsInlineMapping`) to assert `BeOfType<HumlInlineMapping>`.
- Updated `HumlParserTests.Parse_InlineDict_ReturnsMappings` to assert `BeOfType<HumlInlineMapping>` and cast to `HumlInlineMapping`.
- Multiline dict tests (`Parse_MultilineDict_ReturnsMappings`, `Parse_NestedMultilineDicts_ReturnsNestedStructure`) remain `HumlDocument` — `ParseMultilineDict` correctly produces `HumlDocument`.

## Decisions Made

- `HumlInlineMapping` extends `HumlNode` directly (not `HumlDocument`) — keeps type hierarchy flat and unambiguous; consumers can switch over `HumlDocument` vs `HumlInlineMapping` without worrying about inheritance
- Root `{}` case in `RootType.EmptyDict` uses `HumlInlineMapping` as the inner entry in `HumlDocument([HumlInlineMapping([])])` — stays consistent with the new contract
- `RootType.InlineDict` promotes inline mapping entries into `HumlDocument(inlineMapping.Entries)` — root always returns `HumlDocument`; callers of `Huml.Parse()` always receive a `HumlDocument`
- `DeserializeMappingEntries` helper takes `IReadOnlyList<HumlNode>` — both `HumlDocument.Entries` and `HumlInlineMapping.Entries` share the same interface, no downcasting needed
